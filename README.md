# เว็บไซต์ศูนย์ศึกษาพระพุทธศาสนาวันอาทิตย์ วัดสี่แยกสมเด็จ

เว็บแอป Responsive สำหรับนำเสนอคณะกรรมการผ่านแท็บเล็ต/จอสัมผัส พร้อม Admin Dashboard, กราฟ Interactive, สไลด์ภาพ และ Google Apps Script API ที่ใช้ Google Sheets + Drive เป็นฐานข้อมูล

## ฟังก์ชันใหม่ V12

- สำรองข้อมูลอัตโนมัติก่อนบันทึกและก่อนกู้คืน เก็บ 10 ชุดล่าสุด
- กู้คืนข้อมูลจากหน้า “สำรองและประวัติ” พร้อมยืนยันก่อนดำเนินการ
- Activity Log แสดงการบันทึก อัปโหลด และกู้คืน 100 รายการล่าสุด
- ยืนยันก่อนลบ และเตือนเมื่อออกจากหน้าขณะยังไม่ได้บันทึก
- สถานะฉบับร่าง/เผยแพร่สำหรับอัลบั้ม บุคลากร และรางวัล
- ค้นหารายการหลังบ้านด้วยชื่อ ปี หน่วยงาน หรือคำสำคัญ
- ตรวจป้องกันการอัปโหลดไฟล์ชื่อและขนาดเดิมซ้ำ

> เมื่อนำ V12 ไปใช้กับระบบเดิม ให้แทนที่ `Code.gs` แล้วสร้าง Deployment เวอร์ชันใหม่ จากนั้นเปิดหลังบ้านและบันทึกหนึ่งครั้ง ระบบจะสร้างชีต `Backups` และ `Activity_Log` ให้อัตโนมัติ โดยไม่ลบข้อมูลเดิม

## 1) ติดตั้ง Google Apps Script

1. ไปที่ [script.google.com](https://script.google.com) แล้วกด **โครงการใหม่**
2. เปิดไฟล์ `Code.gs` ลบโค้ดเดิม แล้วคัดลอกทั้งหมดจาก `gas/Code.gs` ไปวาง
3. กดบันทึก เลือกฟังก์ชัน `setup` และกด **เรียกใช้**
4. อนุญาตสิทธิ์ Google Sheets และ Google Drive ตามหน้าจอ
5. เปิด **บันทึกการดำเนินการ (Execution log)** จะเห็นลิงก์ Google Sheets, โฟลเดอร์ Drive และรหัสผ่าน Admin เริ่มต้น
6. เปลี่ยนรหัสผ่านทันทีโดยเพิ่มคำสั่งชั่วคราว เช่น `setAdminPassword('รหัสใหม่อย่างน้อย8ตัว')` แล้วกด Run
7. ไปที่ **ทำให้ใช้งานได้ > การทำให้ใช้งานได้รายการใหม่ > เว็บแอป**
8. ตั้งค่า **ดำเนินการในชื่อ: ฉัน** และ **ผู้ที่มีสิทธิ์เข้าถึง: ทุกคน** แล้วกด Deploy
9. คัดลอก URL ที่ลงท้าย `/exec` เก็บไว้

> เมื่อแก้ `Code.gs` ภายหลัง ต้องสร้างเวอร์ชัน Deployment ใหม่ มิฉะนั้นเว็บจะยังใช้โค้ดเวอร์ชันเดิม

## 2) ติดตั้ง Frontend ในเครื่อง

ต้องมี Node.js 20 หรือใหม่กว่า

```bash
npm install
cp .env.example .env
```

เปิด `.env` แล้วแทนค่า URL:

```env
VITE_GAS_API_URL=https://script.google.com/macros/s/DEPLOYMENT_ID/exec
```

จากนั้นทดสอบ:

```bash
npm run dev
```

- หน้าเว็บไซต์: `http://localhost:5173/`
- หลังบ้าน: `http://localhost:5173/#/admin`

ถ้ายังไม่ใส่ URL ระบบจะแสดงข้อมูลตัวอย่างไทยในหน้า Public เพื่อให้ตรวจดีไซน์ได้ แต่ Admin ต้องเชื่อม GAS ก่อน

## 3) Deploy ด้วย Vercel

1. นำโฟลเดอร์นี้ขึ้น GitHub repository
2. เข้า Vercel แล้วเลือก **Add New > Project** และ Import repository
3. Framework Preset เลือก **Vite**
4. เพิ่ม Environment Variable ชื่อ `VITE_GAS_API_URL` และใส่ URL `/exec`
5. กด Deploy
6. เปิด URL ของ Vercel แล้วทดสอบทั้งหน้า `/` และ `/#/admin`

## โครงสร้างฐานข้อมูลที่สร้างอัตโนมัติ

- `Site_Settings` — ชื่อศูนย์ สโลแกน ประวัติ โลโก้ และแบนเนอร์
- `5_Dimensions` — สถิติ กราฟ และรายการภาพของผลการดำเนินงาน 5 ด้าน
- `Timeline` — ประวัติแบบเส้นเวลา ตั้งแต่ พ.ศ. 2428 ถึงปัจจุบัน
- `Quick_Stats` — การ์ดสถิติภาพรวม 4 ใบ
- Google Drive — แยกโฟลเดอร์ภาพตั้งค่าเว็บไซต์และผลงานแต่ละด้าน

## โมดูลหลักฐานและข้อมูลประชาสัมพันธ์

- อัลบั้มโครงการ/กิจกรรม เปิดดูภาพทั้งหมดเป็นสไลด์ได้
- ทำเนียบบุคลากร พร้อมภาพ ตำแหน่ง และรายละเอียด
- รางวัลที่ได้รับ พร้อมภาพหลักฐาน ปี และหน่วยงานที่มอบ
- แผนที่ Google Maps สร้างจากที่อยู่ที่ผู้ดูแลกำหนด

เมื่อเพิ่มอัลบั้ม บุคลากร หรือรางวัลรายการใหม่ ให้กด **บันทึกข้อมูล** หนึ่งครั้งก่อนอัปโหลดภาพของรายการนั้น

## หมายเหตุความปลอดภัย

- รหัสผ่านถูกเก็บเป็น SHA-256 hash ใน Script Properties ไม่ได้บันทึกลง Sheet
- Admin token มีอายุ 6 ชั่วโมงและเก็บชั่วคราวใน Script Cache
- รูปภาพถูกตั้งเป็น “ทุกคนที่มีลิงก์ดูได้” เพื่อให้หน้า Public แสดงผลได้
- สำหรับข้อมูลอ่อนไหวหรือผู้ดูแลหลายคน ควรเปลี่ยนเป็น Google Workspace Sign-In และจำกัดบัญชีตามอีเมล
